# AI 基础概念学习笔记

> 基于 Handbook AI 基础章节的学习总结

---

## 1. LLM（大语言模型）

### 核心定义

LLM（Large Language Model）本质是一个**概率预测模型**。它通过用户输入来预测下一个字或词的输出，核心机制是计算每个词的概率并选择概率最大的词语进行输出。

### 关键概念

| 概念 | 解释 |
|------|------|
| **Token** | 模型处理文本的最小单位，可以是字、词或子词片段 |
| **Embedding** | 将文本、代码或其他对象映射为**向量**（高维数字表示），模型通过这些向量进行概率计算和语义理解 |
| **Transformer** | 现代 LLM 的核心架构，通过自注意力机制捕捉文本中的长距离依赖关系 |
| **Hallucination（幻觉）** | 模型生成看似合理但实际错误或无根据的内容 |
| **Multimodal** | 模型不仅能处理文本，还能理解图像、音频、视频等多种模态的输入 |

### 个人理解

- LLM 不是"理解"内容，而是基于统计概率"预测"最可能的下一个词
- Embedding 是连接人类语言和机器计算的桥梁，相似的语义会有相似的向量表示
- Hallucination 是 LLM 的最大风险之一，需要人工验证关键信息

## 2. Prompt（提示词）

### 核心定义

Prompt 是用户输入给模型的文字指令。通过**优化提示词的输入**，可以引导模型获得更优质、更符合预期的输出。

### 关键要点

- **Prompt Engineering**：设计和优化提示词以获得更好输出的技术
- **System Prompt**：设定模型角色和行为边界的基础指令
- **User Prompt**：用户的具体问题和请求

### ⚠️ 安全风险：Prompt Injection（提示词注入）

- 类似于传统的 **SQL 注入攻击**
- 攻击者通过在输入中注入恶意指令，让模型执行非预期的操作
- 防御措施：输入过滤、输出校验、权限隔离

### 个人理解

- Prompt 是与 LLM 交互的主要接口，好的 Prompt 能显著提升输出质量
- 安全边界很重要，特别是在 Agent 和工具调用场景中

## 3. Context（上下文）

### 核心定义

Context 是模型在一次对话或任务中能够"看到"和使用的信息范围。模型的理解和输出质量高度依赖上下文的质量和长度。

### 关键概念

| 概念 | 解释 |
|------|------|
| **Context Window** | 模型一次能处理的最大文本长度（token 数） |
| **Memory** | 跨 Session 保存的知识，在不同对话中持续可用 |

### 实践要点

- **上下文压缩**：当上下文达到窗口的 **60%** 左右时，模型性能开始下降
- **压缩策略**：
  - 去除冗余信息
  - 摘要历史对话
  - 使用外部存储（如 Vector DB）补充上下文

### Claude Code 中的 Memory

- 记录在 `CLAUDE.md` 文件中的信息
- 跨 Session 持久化保存
- 用于存储项目规范、代码风格、常用命令等

### 个人理解

- 上下文管理是 AI 应用开发的核心技能
- 不是给模型越多信息越好，而是给"越相关的信息越好"
- 需要主动管理上下文，避免信息过载

## 4. RAG（检索增强生成）

### 核心定义

RAG（Retrieval-Augmented Generation）是一种将**外部知识检索**与**语言模型生成**结合的技术。让模型在生成回答前先检索相关文档，从而提供更准确、可追溯的回答。

### 工作流程

```
用户问题 → Chunking（切分文档） → Embedding（向量化） 
→ Vector DB（存储） → Retriever（检索） → Rerank（重排序） 
→ 上下文注入 → LLM 生成回答 → Citation（引用来源）
```

### 关键组件

| 组件 | 功能 |
|------|------|
| **Chunking** | 将长文本切分成适合处理的小块 |
| **Vector DB** | 存储文档的 embedding 向量，支持语义检索 |
| **Metadata** | 存储额外信息：时间、来源、是否废弃等 |
| **Retriever** | 根据用户问题从数据库中检索相关文档 |
| **Rerank** | 对初步检索结果重新排序，提高准确性 |
| **Citation** | 将检索结果链接回原始数据来源，确保可追溯 |

### 检索策略

1. **先过滤再检索**：先用 metadata 过滤，再在过滤后的范围内检索
2. **混合检索**：结合关键词检索和语义检索
3. **Rerank 场景**：候选结果很多、相似度接近、技术文档标题相近但内容不同

### 个人理解

- RAG 是解决 LLM "幻觉"和"知识过时"的有效方案
- Chunking 策略直接影响检索质量：切太大可能包含冗余，切太小可能丢失上下文
- Citation 不仅是用户体验问题，更是**可验证性**和**信任**的基础

---

## 总结

| 概念 | 一句话总结 |
|------|-----------|
| LLM | 基于概率预测下一个词的大模型 |
| Prompt | 与模型交互的指令，需要优化和安全防护 |
| Context | 模型可见的信息范围，需要主动管理和压缩 |
| RAG | 检索+生成，让模型有"参考书"可查阅 |
