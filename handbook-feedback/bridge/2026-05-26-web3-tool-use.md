# AI × Web3 Bridge 学习笔记：Web3 Tool Use（Web3 工具调用）

> 基于 Handbook AI × Web3 Bridge 章节  
> Agent 不能只靠模型文本输出，需要工具去读取链上状态并执行操作

---

## 0. 为什么需要 Web3 工具

### 核心原则

AI Agent 与 Web3 结合的场景中，**不可只靠模型文本输出**，需要使用工具直接与链交互。

### Web3 工具全景

| # | 工具 | 用途 |
|---|------|------|
| 1 | **RPC Tool** | 读取链状态、查询区块、估算 Gas |
| 2 | **Contract Read** | 调用合约 view/pure 函数（只读） |
| 3 | **Contract Write** | 改变链上状态的写入操作 |
| 4 | **Wallet Tool** | 连接账户、请求签名、管理授权 |
| 5 | **Explorer Tool** | 交易后查询验证 |
| 6 | **DeFi Tool** | Swap、借贷等资产操作封装 |
| 7 | **Tool Permission** | 定义 Agent 可调用的工具和条件 |
| 8 | **Tool Log** | AI Agent 的可审计基础 |

---

## 1. RPC Tool（远程过程调用工具）

### 核心定义

**RPC Tool** 用于读取链状态、查询区块、估算 Gas、获取日志和广播交易。

### RPC Tool 能做什么

| 操作 | 说明 |
|------|------|
| **读取链状态** | 查询余额、Nonce、合约存储 |
| **查询区块** | 按高度或哈希获取区块信息 |
| **估算 Gas** | 交易前预估费用 |
| **获取日志** | 获取合约 Event 日志 |
| **交易广播** | 将签名后的交易广播到链上 |

---

## 2. Contract Read（合约读取）

### 核心定义

**Contract Read** 用于调用合约的 **view** 和 **pure** 函数——**不改变链上状态**，不需要 Gas。

### 读取的内容

```
- 代币余额 (balanceOf)
- 总供应量 (totalSupply) 
- 池子状态 (getReserves)
- 利率 (getAPR)
- 权限查询
- 任何只读的公开数据
```

### 特点

| 特性 | 说明 |
|------|------|
| **Gas 消耗** | 无（只读调用） |
| **状态变更** | ❌ 不改变链上状态 |
| **签名要求** | ❌ 无需用户签名 |
| **可缓存** | ✅ 可缓存，但注意数据新鲜度 |

---

## 3. Contract Write（合约写入）

### 核心定义

**Contract Write** 是**会改变链上状态**的操作。这是 Agent 真正 "做事" 的工具——发送交易、调用合约函数。

### ⚠️ Contract Write 的安全检查清单

| 检查项 | 说明 |
|--------|------|
| **Chain ID** | 确认当前网络，防止跨链错误 |
| **目标合约地址** | 确认调用的是正确的合约 |
| **ABI Method + 参数** | 函数签名和参数类型匹配 |
| **Value / Token 变化预估** | 会转出多少 ETH/Token？收到什么？ |
| **Gas 估算** | 预估费用 |
| **模拟结果** | 预演执行是否成功 |
| **Policy 检测** | 是否符合权限策略（额度、白名单等） |
| **用户或 Smart Account 授权** | 必须有用户签名或智能账户授权 |
| **交易哈希 + 回执追踪** | 提交后追踪执行结果 |

### 执行流程

```
用户/Agent 发起意图
    ↓
  1. 读取 Contract ABI
  2. 编码函数参数
  3. Gas 估算
  4. ✅ 交易模拟 → 验证结果
  5. Policy 检测 → 检查权限边界
  6. 用户签名授权 (或 Smart Account)
  7. 广播交易到链上
  8. 追踪交易回执
  9. 返回执行结果 + Citation
```

---

## 4. Wallet Tool（钱包工具）

### 核心定义

**Wallet Tool** 负责连接账户、请求签名、生成交易、管理授权并返回用户确认结果。

### 功能

| 功能 | 说明 |
|------|------|
| **连接账户** | 连接用户的钱包（MetaMask、WalletConnect 等） |
| **请求签名** | 请求用户对交易或消息签名 |
| **生成交易** | 组装交易数据（to、value、data 等） |
| **管理授权** | 管理 spending allowance、Session Key |
| **返回确认结果** | 告诉 Agent 用户是批准还是拒绝 |

---

## 5. Explorer Tool（区块浏览器工具）

### 核心定义

**Explorer Tool** 用于在交易达成后查询验证——查询交易详情、合约源码、Event、Token Transfer 和历史地址。

### 用途

| 用途 | 说明 |
|------|------|
| **验证交易是否成功** | 查看交易状态、Gas 消耗 |
| **查看合约源码** | 验证合约逻辑是否与文档一致 |
| **查看 Event** | 确认操作触发了哪些事件 |
| **查看 Token Transfer** | 确认资产确实已转移 |
| **历史地址分析** | 查看地址的过往交互记录 |

### 在 Agent 工作流中的位置

```
交易前:  RPC Tool + Contract Read + Simulation
交易中:  Wallet Tool + Contract Write
交易后:  Explorer Tool ← 验证 + 生成 Citation
```

---

## 6. DeFi Tool（DeFi 工具）

### 核心定义

**DeFi Tool** 把 Swap、借贷、授权、仓位查询等能力封装给 Agent。

### 封装的典型操作

| 操作 | 涉及协议 | 风险等级 |
|------|---------|---------|
| **Swap** | Uniswap、Curve | 🟡 中 |
| **Lending (存款/借款)** | Aave、Compound | 🟡 中 |
| **授权 (Approve)** | ERC-20 | 🔴 高 |
| **仓位查询** | 各协议 | 🟢 低 |
| **质押 (Stake)** | Lido、Rocket Pool | 🟡 中 |
| **收益优化** | Yearn、Pendle | 🔴 高 |

### ⚠️ 安全问题

> DeFi 工具涉及真实资产，需要**特别谨慎**。每次调用前必须：
> - 模拟执行
> - 检查 Policy
> - 清晰的用户确认
> - 记录完整的 Tool Log

---

## 7. Tool Permission（工具权限）

### 核心定义

**Tool Permission** 定义 Agent 可以调用哪些工具，在什么条件下调用。

### 权限控制维度

| 维度 | 示例 |
|------|------|
| **白名单** | Agent 只能调这 5 个协议 |
| **额度限制** | 每笔不超过 100 USDC，每天不超过 1000 USDC |
| **时间约束** | 只能在 9:00-21:00 操作 |
| **用户确认** | 写操作必须用户签名 |
| **策略引擎** | 根据交易风险评估是否允许自动执行 |

### 权限分级

```
只读层:  RPC Tool, Contract Read, Explorer Tool
  → 无需签名，可自动执行

交易层:  Contract Write, DeFi Tool
  → 需要模拟 + Policy 检测 + 用户签名

高敏层:  Approve, Upgrade, Transfer 所有权
  → 需要最高级别的确认（多签 / 时间锁）
```

## 8. Tool Log（工具日志）

### 核心定义

**Tool Log** 是 AI Agent 的**可审计基础**。每条工具调用都应该留下完整的日志记录。

### 日志必须记录的信息

| 字段 | 说明 |
|------|------|
| **用户目标** | 用户原始意图（"我想把 100 USDC 换成 ETH"） |
| **工具名** | 调用了哪个工具（SwapTool） |
| **参数** | 完整的输入参数 |
| **输出结果** | 工具返回的结果 |
| **错误信息** | 如果有失败，记录错误详情 |
| **时间戳** | 调用发生的链上时间或系统时间 |
| **会话 ID** | 关联到哪一次对话/任务 |
| **用户确认状态** | 用户是批准还是拒绝 |

### 日志的价值

```
短期:  调试和排错
中期:  安全审计、异常检测
长期:  Agent 行为分析和改进
```

```
                      ┌─────────────────────┐
                      │   用户意图 / Prompt   │
                      └──────────┬──────────┘
                                 │
                    ┌────────────▼────────────┐

                    └────────────┬────────────┘
                                 │
                    ┌────────────▼────────────┐
              ┌─────┤   读取链上状态 RPC      ├─────┐
              │     │   Contract Read        │     │
              │     └────────────┬────────────┘     │
              │                  │                  │
              ▼                  ▼                  ▼
       ┌────────────┐   ┌──────────────┐   ┌────────────┐
       │ Simulation  │   │  DeFi Tool   │   │ Wallet Tool│
       │ (模拟执行)   │←──│  (Swap/借贷)  │   │ (签名授权)  │
       └────────────┘   └──────────────┘   └────────────┘
              │                  │                  │
              └──────────────────┼──────────────────┘
                                 │
                    ┌────────────▼────────────┐
                    │  Contract Write         │ ← 写入链上
                    │  (签名 + 广播 + 回执追踪) │
                    └────────────┬────────────┘
                                 │
                    ┌────────────▼────────────┐
                    │  Explorer Tool          │ ← 交易后验证
                    │  (验证 + Citation 生成)   │
                    └────────────┬────────────┘
                                 │
                    ┌────────────▼────────────┐
                    │     Tool Log            │ ← 全链路审计
                    │  (每一步都留下可查记录)    │
                    └─────────────────────────┘
```
