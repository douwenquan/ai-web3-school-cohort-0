## 0. DeFi 是什么

### 核心定位

DeFi（Decentralized Finance）是当前 Web3 最主流、热度最高的应用方向。它在合约系统中管理传统金融的核心状态。

### 传统金融 vs DeFi

| | **传统金融** | **DeFi** |
|------|------|------|
| 余额 | 银行账本 | 合约内记录 |
| 抵押 | 机构审批 | 智能合约自动执行 |
| 债务 | 信用体系 | 超额抵押 + 链上清算 |
| 流动性 | 做市商 + 订单簿 | AMM 流动性池 |
| 交易 | 交易所撮合 | 用户直接与合约交互 |
| 清算 | 人工/机构处理 | 合约自动触发 |

## 1. Token（代币）

### 核心定义

Token 是 DeFi 的**资产单位**。不同类型的 Token 有不同的标准和功能。

### ERC-20：同质化代币标准

**ERC-20** 是最常见的同质化 Token 标准。同质化意味着每个 Token 完全一样、可以互换（就像 1 美元和另一个 1 美元）。

| 功能 | 方法 | 说明 |
|------|------|------|
| 查询余额 | `balanceOf(address)` | 查看某地址持有多少 |
| 转账 | `transfer(to, amount)` | 向另一个地址转账 |
| 授权 | `approve(spender, amount)` | 允许其他合约使用你的 Token |
| 授权转账 | `transferFrom(from, to, amount)` | 被授权方代为转账 |
| 总供应量 | `totalSupply()` | 该 Token 的总发行量 |

### 其他 Token 标准

| 标准 | 类型 | 示例 |
|------|------|------|
| **ERC-20** | 同质化代币 | USDT, UNI, LINK |
| **ERC-721** | 非同质化代币 (NFT) | CryptoPunks, BAYC |
| **ERC-1155** | 多代币标准（同质化+非同质化） | 游戏资产 |
| **ERC-4626** | 收益代币化标准 | 生息 Token Vault |

## 2. AMM（自动化做市商）

### 核心定义

**AMM（Automated Market Maker）** 用**流动性池 + 定价公式**替代传统订单簿，让用户可以直接和合约交易，无需等待对手方。代表项目是 **Uniswap**。

### 工作原理

```
传统订单簿:
  买方出价 $100 → 等待 → 卖方报价 $101 → 撮合成交

AMM (Uniswap):
  用户A 存入 ETH + USDC → 流动性池
  交易者: 用 ETH 换 USDC → 价格由池子状态和公式决定
```

### Uniswap 的恒定乘积公式

```
x × y = k

  x = Token A 在池中的数量
  y = Token B 在池中的数量
  k = 恒定乘积

交易者用 Δx 个 A 换 B:
  池中 A 变多 → x 变大 → y 必须变小 → B 变贵
  交易越大 → 价格越差
```

### ⚠️ AMM 的四个关键风险

#### 2.1 Slippage（滑点）

**预期价格和实际成交价格的差距**。

```
你看到: 1 ETH = 2000 USDC
实际成交: 1 ETH = 1980 USDC

滑点 = (2000 - 1980) / 2000 = 1%
```

原因：由于 AMM 公式，交易会改变池子比例，导致价格在交易过程中移动。

#### 2.2 LP（流动性提供者）+ 无常损失

**LP** 向池子存入两种资产，赚取交易手续费。但需要承担**无常损失（Impermanent Loss）**。

```
你存入: 1 ETH + 2000 USDC
池子总价值: 你的份额 = y%

如果 ETH 涨到 3000 USDC:
  你的份额价值 = 等比例提取的价值
  如果直接持有 1 ETH + 2000 USDC = 5000
  从池子提取 ≈ 4899
  
  无常损失 ≈ 101 USDC
  
只要价格回到存入时的水平，无常损失消失 → 所以叫"无常"
```

#### 2.3 Price Impact（价格影响）

大额交易会改变池子比例，导致成交价格显著变差。

```
小额交易 → 池子比例几乎不变 → 价格接近市价
大额交易 → 池子比例大幅变化 → 成交价远差于市价
```

#### 2.4 MEV 风险（Sandwich Attack）

交易排序可能被利用。

```
1. 攻击者看到你的大买单 → 在你之前买入 → 拉高价格
2. 你的交易成交 → 以被拉高的价格成交（更贵）
3. 攻击者在你的交易之后卖出 → 赚取差价

这就是 Sandwich Attack（三明治攻击）
```

## 3. Lending（借贷）

### 核心定义

借贷协议把**存款、借款、抵押率、利率、清算规则**写到合约里。用户可以存入资产赚取利息，也可以抵押资产借出另一种资产。

### 借贷流程

```
存款方:  存入 100 USDC → 赚取利息
借款方:  抵押 1 ETH → 借出 500 USDC → 支付利息

利率由资金利用率决定:
  借的人多 → 利率上升 → 吸引更多存款
  借的人少 → 利率下降
```

### 关键概念

| 概念 | 说明 |
|------|------|
| **抵押率 (Collateral Ratio)** | 借出金额 / 抵押品价值。如 150% 抵押率：存 1.5 ETH 才能借等值 1 ETH 的 USDC |
| **健康因子 (Health Factor)** | 抵押品价值 / 借款价值。低于 1 → 面临清算 |
| **清算** | 抵押品价值不足时，合约自动卖出抵押品偿还借款 |
| **清算惩罚** | 被清算方需要额外支付罚金（如 +5%） |

### 清算流程

```
1. 你存 1 ETH ($3000) 作为抵押
2. 借出 2000 USDC（抵押率 150%）
3. ETH 跌到 $2200
4. 健康因子 < 1
5. 清算人触发清算 → 用 USDC 买走你的 ETH（有折扣）
6. 你的 ETH 被卖出，借款被偿还，剩余 ETH 退回
7. 清算人赚取差价和罚金
```

## 4. Stablecoin（稳定币）

### 核心定义

Stablecoin 是 DeFi 里的**计价和结算基础**，目标是维持与法币（通常是 USD）1:1 的锚定。

### 维持稳定的不同机制

| 类型 | 机制 | 代表 | 稳定性 |
|------|------|------|--------|
| **法币储备型** | 1 USDC = 银行存 1 USD + 国债收益 | USDC, USDT | 🟢 高 |
| **加密超额抵押型** | 存入 $1.5 ETH → 生成 $1 DAI | DAI (MakerDAO) | 🟡 较高 |
| **算法型** | 算法调节供需，无充分抵押 | UST (已崩盘) | 🔴 不稳定 |
| **混合型** | 多种抵押品 + 部分算法 | FRAX | 🟡 中等 |

### 看稳定币时必须问的 5 个问题

| 问题 | 为什么要问 |
|------|-----------|
| **靠什么维持锚定？** | 法币储备 vs 算法 — 机制决定底线 |
| **谁可以发行和销毁？** | 中心化 vs 去中心化 — 决定信任模型 |
| **储备和抵押品在哪里？** | 银行、链上合约、还是不可验证的 |
| **脱锚时谁承担损失？** | 持有者、LP、还是协议本身 |
| **主要池子流动性有多深？** | 大额兑换会不会引发死亡螺旋 |

### ⚠️ 历史上重要的脱锚事件

- **UST/LUNA (2022)**：算法稳定币，死亡螺旋中归零，数百亿美元蒸发
- **USDC (2023)**：硅谷银行倒闭，USDC 一度脱锚到 $0.87，恢复后验证了储备透明度的重要性

## 5. Liquidity（流动性）

### 核心定义

**流动性决定资产能不能以合理价格买入、卖出、借出、清算或退出。**

### 流动性为什么重要

```
有流动性:  想卖 100 ETH → 池子够深 → 价格 $3000 → 成交 ✅
无流动性:  想卖 100 ETH → 池子太浅 → 价格 $500 → 成交 ❌

DeFi 中的价格如果没有流动性支撑，就只是一个数字。
```

### 流动性在 DeFi 各环节的作用

| 环节 | 流动性不足的后果 |
|------|-----------------|
| **交易** | 大额交易产生巨大滑点 |
| **借贷** | 清算时抵押品卖不出去，坏账积累 |
| **稳定币** | 脱锚时抛售导致踩踏，加速死亡螺旋 |
| **退出** | 牛市高点账面盈利，实际卖不掉 |

## DeFi 全景总结

```
Token (ERC-20)
    │
    ├── AMM (Uniswap)
    │   ├── 恒定乘积公式 (x × y = k)
    │   ├── LP 提供流动性 → 赚手续费 + 承担无常损失
    │   ├── 滑点 / 价格影响
    │   └── MEV 风险 (Sandwich Attack)
    │
    ├── Lending (Aave, Compound)
    │   ├── 存款赚利息 / 抵押借款
    │   ├── 抵押率 + 健康因子
    │   └── 自动清算
    │
    ├── Stablecoin
    │   ├── 法币储备型 (USDC/USDT)
    │   ├── 加密超额抵押型 (DAI)
    │   └── 算法型 (已证明高风险)
    │
    └── Liquidity ← 一切价格的前提
```
